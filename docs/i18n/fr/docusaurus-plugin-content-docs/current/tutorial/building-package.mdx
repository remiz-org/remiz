# Construction d'un package
Cette page décrit le processus de consutrction d'un package et comment constuire
votre premier package.

## Processus de build/construction
Pour chaque packager déclaré dans votre configuration de package, Remiz appelera
séquentiellement le point d'entrée de vos packagers (tel que défini dans la
configuration globale).
Pour chaque packager, Remiz récupèrera les fichiers de ces dossiers magiques.
Remiz peut enregistrer 0, 1 ou plusieurs fichiers issus du même packager mais
un avertissement sera affiché si aucun fichier n'est trouvé.

:::caution Caution

Si un packager sort avec un code retour différent de zéro, Remiz arrêtera le
processus de construction du package.

:::


Après que tous les packagers soient traités, Remiz produira le package final en
incluant tous les fichiers récupérés et ajoutera de la métadonnée.


## Commande build
Pour démarrer la construction d'un package, appelez simplement la sous-commande
`build` :

`remiz build path/to/package_config.toml`

Vous pouvez également inclure la position du fichier de configuration globale
à utiliser et passer des arguments supplémentaires aux packagers :

`remiz build path/to/package_config.toml -g path/to/global_config.toml -- --extra args`

## Vérifiez vos packages .pack
Une fois que votre package a été construit correctement, vous devez l'inspecter
avant de considérer votre pipeline stable.

Vous pouvez utiliser la commande `unpack` à cet effet.
