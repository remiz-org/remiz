## Vérifiez votre déploiement

Après que vous ayez déployé votre premier package, il est vivement recommandé
de vérifier que toutes les parties de votre projet ont correctement été déployés
selon vos règles.

:::tip Tip

Une façon pour vérifier ce bon fonctionnement est de faire une sauvegarde de votre
projet. De construire le package de ce projet puis de supprimer le projet.
Il ne reste plus qu'à le déployer et reconstruire un package à partir de ce projet
nouvellement déployé. Si tout s'est bien passé, la commande `remiz diff` ne devrait
afficher aucune différence.

:::
